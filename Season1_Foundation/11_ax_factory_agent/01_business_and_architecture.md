# 🏭 제조 현장의 AI Transformation (AX) 기획과 아키텍처

## 1. 왜 제조업에 챗GPT를 그대로 쓸 수 없는가?
빅테크의 챗GPT는 전 세계의 일반적인 지식을 알고 있지만, **우리 공장의 '3번 CNC 선반'이 2년 전에 어떤 부품으로 교체되었는지**는 절대 모릅니다. 또한, 제조 데이터는 국가 핵심 기술이거나 사내 기밀이므로 외부 클라우드로 데이터를 넘길 수 없습니다.
➔ **해결책:** 사내망(On-Premise)에 작은 오픈소스 LLM(Small Language Model)을 올리고, 사내 문서만 보안 공간에서 검색하는 **보안형 RAG 아키텍처**가 필수적입니다.

## 2. AX-Factory-Agent의 구동 시나리오
단순히 "E-04 에러는 모터 고장입니다"라고 대답하는 것은 진정한 AX가 아닙니다. 작업자가 진짜 원하는 것은 **문제 해결**입니다.

*   **AS-IS (현재):**
    1. 기계 정지 (E-04 발생)
    2. 작업자가 500p짜리 종이 매뉴얼 뒤적임 (30분 소요)
    3. 모터 고장임을 확인하고, 자재과에 전화를 걸어 모터 재고 있는지 물어봄 (15분 소요)
    4. 정비 반장님께 수리 요청 수기 작성 및 전달 (10분 소요)
    ➔ **총 55분의 Downtime 발생**
*   **TO-BE (Agent 도입 후):**
    1. 기계 정지 (E-04 발생)
    2. 작업자가 음성 또는 텍스트로 *"E-04 에러 떴어, 해결해줘."* 입력.
    3. **[Agent 작동]** RAG 검색 ➔ "모터 고장" 인식 ➔ 재고 DB API 자동 찔러봄 ➔ 재고 2개 확인 ➔ 정비 티켓 시스템 API 자동 찔러서 티켓 발행.
    4. 3초 후 Agent 응답: *"E-04 모터 결함입니다. 자재 창고에 재고 2개 확인되어 정비 부서에 교체 지시(Ticket #102) 발행 완료했습니다."*
    ➔ **총 3초의 Downtime, 그리고 인건비 제로.**

이것이 단순한 LLM 튜토리얼을 넘어선, **기업의 돈을 벌어주는 진짜 AI 엔지니어링**입니다.
